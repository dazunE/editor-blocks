Thoughts on a Visual Editor Redesign

A text-first content block approach.

Please keep in mind that these are just my personal ideas and opinions, nothing close to something final.

A Note on Accessibility

I hope a redesign of the editor experience to have accessibility strongly in mind. Accessibility comes hand in hand with creating something usable, whether you interact with it visually, on desktop or mobile, with the keyboard, voice, or any other input and output method. Thinking about how and when to present information and interaction methods should come first, and be independent of the type of user interface, which helps the experience become more aligned, similar and beneficial to all users. It might even make it easier to develop.

While thinking about a visual redesign for the visual editor, I've found people are trying to solve very similar issues for, e.g., screen readers. Thinking from such a perspective can trigger realisations that are useful for visual interfaces too.

With this in mind, let's go over some possible scenarios.

There are roughly three kind of blocks.

Object Blocks

  • They cannot contain any other content blocks.
  • Ideally these blocks should be alignable.
  • Examples: images, galleries, audio, video, iframes, horizontal rules.

This type of block is arguably the easiest to handle. Interactions with these can usually be split in two: default interactions (like playing a video) and actions that change the object based on the available options.

Similarly, if I use a screenreader, I expect there options to be available for both interacting with the video and altering it. That last option may or may not need to be wrapped in an extra step if too many controls are available. E.g. "Press X to access controls to alter this video."

Ideally, any controls added to objects inside the editor should be in relation to the other content. Maybe options to align or move an image should be given, but not to edit the image itself. I think most objects could therefore have the exact same controls: align, move, edit, and remove. Some blocks could choose to have nested elements though, such as galleries, so maybe in these cases one should be allowed to alter the composition within the editor.

Today, WordPress already handles some of these object blocks quite well by showing a contextual toolbar.
